## Domain Terms

- **Report** — a self-contained `.html` file generated by the `html-report` skill after a Stage 4 review skill completes; contains scored dimensions and a findings table.
- **Report Slot** — a `{{PLACEHOLDER}}` in `templates/report_template.html` that the LLM fills during rendering (e.g. `{{RISK_SCORE}}`, `{{FINDINGS_ROWS}}`).
- **Scoring Dimension** — one axis of evaluation shown as a 0–100% progress bar: Risk, Code Quality, or Adaptation Effort.
- **Report Session** — one invocation of a Stage 4 skill; produces one report file at `reports/<skill>_<branch>_<ISO-timestamp>.html`.
- **Thinking Report** — a self-contained `.html` file generated by the `thinking-report` skill after a Stage 0.5–2 session locks a direction; contains a Decision box, Trade-Off Matrix, and Assumptions list.
- **Trade-Off Matrix** — an options × criteria HTML table with ✅/⚠️/❌ cells; the chosen option column is highlighted green (`col-chosen` CSS class).
- **Thinking Session** — one invocation of brainstorming, grilling, or planning that produces a direction lock; triggers one `thinking-report` at `reports/thinking-report_<branch>_<ISO-timestamp>.html`.

- **Pack** — an optional domain extension bundle: `agents/` + `skills/` + `PACK.md`. Installed via `setup.sh --pack=<name>`; symlinked into `.claude/agents/` and `.claude/skills/` alongside core resources.
- **Core framework** — the always-installed base: backend, frontend, common-infrastructure, qa agents + all `.claude/skills/` entries. Packs are additive on top; they never replace core resources.
- **Pack agent** — a domain-specific sub-agent defined in `packs/<name>/agents/<agent-name>.md`. Uses a namespaced name (e.g. `mobile-developer`) to avoid collision with core agents.
